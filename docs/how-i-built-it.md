## セッション1：「FLOW」が生まれた日（Session 872fad1e）

### ステップ1：市場分析から始まった

最初のプロンプトがこれです：

```
どんなアニメだったら今一番流行るか分析して教えてください
```

Claudeの回答は「異能バトル×現代社会問題のハイブリッド」「弱者の戦略家型主人公」「謎解き+伏線回収の設計」など。

そこで追加で聞きました：

```
柔術要素は？
```

これに対してClaudeが返してきた分析：

> **「柔術アニメが当たる根拠」**
> - BJJ主軸のアニメはゼロ — 完全ブルーオーシャン
> - 寝技の攻防は「チェスのような頭脳戦」として描ける
> - 帯色がそのままランク制と完全一致
> - 海外BJJ人口が急成長中

### ステップ2：ヒット作品の要素分解

```
ヒット作品かなんがえてまずは要素から
```

Claudeが進撃/鬼滅/呪術/フリーレンなど近年のメガヒット作品を横断分析して「7つの必須要素」を抽出。

### ステップ3：「FLOW」が生まれた一言

```
お願いします。オリジナルも付け加えて。お願い。
```

**これだけで「FLOW」の原型が出てきました。**

Claudeが返してきたのが：

> **タイトル: 「FLOW」（フロウ）**
>
> コンセプト: 「身体のチェス」×「AIが支配する格闘技界」×「失われた師匠の技」
>
> 1話のアバン: 師匠が「AIに一切頼らない読めない柔術」で世界王者を破る。直後、師匠が失踪する。
>
> 主人公の欠落: 身体の左半分の感覚が鈍い（幼少期の事故）。右半身だけで組む異形のガードが逆にAIの予測から外れる。
>
> 独自システム: 技と技の「間」に入る意識状態「フロウ」— AIが分節できない連続する水の流れ。

市場分析 → ジャンル絞り込み → 要素抽出 → 一言でオリジナル化、という3ステップ。
プロンプトは合計4つ、所要時間は数十分でした。

168cm, 62kg, messy black hair falling over left eye,

```

初めての人もメールアドレスとか登録しちゃうような感じにしてほしいです
モバイルも最適化した上で見やすいサイトにしてください
```

```

ファビコンとかOGPとかも完璧にしてね
```

→ ここでClaudeが自分でサイトをレビューして問題をリストアップ：

> 1. favicon なし（ブラウザタブが空白）
> 2. OGP画像が3:4比率（1200x630推奨）
> 3. X が `@flow_anime_jp` のまま（`@flow_bjj` に変更必要）
> 4. 多言語対応なし
> 5. JSON-LD構造化データなし
> 6. canonical URLなし
> 7. メール登録がイベントログに混在（専用テーブルなし）
> 8. track.jsのバグ

全部並列で修正→デプロイ。

```
独自にアクセス解析も入れて、ダッシュボードも管理者用に作って、
また英語、日本語、ポルトガル語に完璧に対応してください
```

→ これで `dashboard.html`（管理者用）と3言語対応が生まれました。

```
https://x.com/flow_bjj Xはこれどう？
```

→ 全箇所を `@flow_bjj` に統一。

```
滞在時間とかもわかるようにしてめ
```

→ server.py にタイムゾーン別・ページ別の滞在時間集計が追加されました。

## セッション3：ストーリーの厳しめレビュー（Session 37c36b8b）

サイト公開後に `flow-anime/` ディレクトリに入ってセッションを開始。

```
このプロジェクトを見て改善する部分もっと面白くする部分含めどんなことができるか考えて教えてください
```

```
ストーリーのところはどう？具体的にfbちょうだい厳し目に
```

これに対するClaudeの回答が痛烈でした：

> **致命的な問題点 その1: 「AI支配」の設定が飾りになっている**
>
> 「予測精度99.8%のAIが支配する」→ なのに流は「左腕が壊れてるからAIに読まれない」
>
> これだと「AI支配」は単なる背景設定で、物語の本質に関与していない。
> 「左腕が壊れてるから勝てる」は「ハンデが武器」という古典的パターン。
> 2030年である必然性が薄い。2024年でも成立する話。
>
> **致命的な問題点 その2: 流の動機が弱すぎる**
>
> 主人公の動機が「先生に会いたい」だけだと17歳の少年としては受動的すぎる。

```
やってみて
```

この一言で、Claudeがストーリーを再構築。
`story/episode_guide_s1_revised.md` が生成されました。

```
変更分githubでみせて
```

## 気づいたこと

### プロンプトは短くていい

「お願いします。オリジナルも付け加えて。お願い。」
「全部一kに」（一気に、のタイポ）
「geminidewebで」（Geminiで、のタイポ）

タイポがあっても意図は伝わります。
長い指示よりも、**「何をしたいか」のコアが明確**なほうが大事。

### 「並行して」が最強のワード

「キャラクターシート5枚作ってほしい、それと**並行して**ストーリーに必要なものは作っていってほしい」

これを言った瞬間にClaudeがサブエージェントを複数起動して並列処理を始めます。
時間が1/4になります。

### 「ダメ出しして」が品質を上げる

自分でダメ出しを求めるプロンプトが複数あります：
- 「具体的にfbちょうだい厳し目に」
- 「さらにダメ出しをして、宣伝させて、デプロイして」

自己批評→修正→デプロイのサイクルを回すことで、
一人でもクオリティコントロールができます。

### 人間がやったこと

- 「柔術アニメが見たい」という最初の欲求
- 「マルセロガルシアって実在するw」という知識
- 「ドラゴンボールみたいな」というビジョン
- CAPTCHAを解く
- 出来上がったものを見て「違う」と判断する感覚

すべての判断の最後は人間。
AIは「実行速度」を上げる道具で、「センス」は人間のものです。
